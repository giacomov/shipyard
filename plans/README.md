# plans/

This directory holds implementation plan files generated by the `shipyard plan` command. Each file is named `i<issue-number>.md` and corresponds to a GitHub issue.

## Test fixtures

**Do not remove `i17.md`.** It is a fixture required by the plan-driver workflow test (case 2: review feedback). That test scenario assumes a plan already exists on the branch — `i17.md` stands in for the file that case 1 would normally push to the remote. Removing it will cause case 2 to fail with `FileNotFoundError`.
